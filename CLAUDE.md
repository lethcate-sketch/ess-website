# CLAUDE.md

このファイルは Claude Code がセッション中ずっと参照する常時ルールです。
詳細な仕様は必ず `PROJECT_SPEC.md` を参照してください。本ファイルは「絶対に守る制約」の要約です。

## プロジェクト概要
英語ディスカッションサークルの公式ウェブサイト。広報サイト（公開）と管理システム（要ログイン）の2役割を持つ。
スタック: Next.js(App Router/TS) + Tailwind CSS / Flask(Python) API / Prisma + SQLAlchemy / SQLite(開発)→PostgreSQL(本番)。

## 最優先の制約（厳守）
- 設計の真実源は `PROJECT_SPEC.md`。判断に迷ったら必ず参照し、勝手に別構成へ変えない。
- Prisma がスキーマの真実源。マイグレーションは Prisma Migrate で行う。
- SQLite は enum 非対応のため、enum は使わず `String` 型 + デフォルト値で表現し、許可値はコメントとアプリ層バリデーションで管理する。
- Flask は SQLAlchemy で同一DBにアクセスし、テーブル名・列名を Prisma の `@@map`/`@map` と一致させる。
- 認証JWTは httpOnly Cookie に格納し、Next.js の Route Handler(BFF)経由で Flask に中継する。トークンをブラウザのJSに晒さない。
- 読み取り中心の公開ページ・一覧は Next.js が Prisma Client で直接DB参照。書き込み・業務ロジックは必ず Flask API 経由。
- DB列/テーブルは snake_case、TS/Prismaモデルフィールドは camelCase。
- 日時はUTCで保存し、表示時にクライアントでローカル変換する。

## 確認を取るべきケース
- 仕様の解釈に迷う、または仕様間で矛盾を見つけたときは、勝手に判断せず確認する。
- `.env` の機密値（JWT_SECRET、初期管理者の認証情報、DB接続文字列など）は自分で決めず、ユーザーに入力を促す。
- 破壊的な操作（既存ファイルの大量削除、マイグレーションのリセット等）の前に確認する。

## セキュリティ
- パスワードは bcrypt 等でハッシュ化して保存する。平文を保存しない。
- 機密値を `.env.example` やコミット対象に書き込まない（プレースホルダのみ）。
- 管理者専用APIは role=ADMIN を必ず検証し、権限不足は 403 を返す。
